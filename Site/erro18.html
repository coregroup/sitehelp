Uso incorreto das chaves em comandos de repetição



Um erro que acontece quando esquecemos ou colocamos em um lugar errado as chaves de um comando de repetição, seja no início ou fim.


Exemplos:

ERRADO

 
//Descricao do programa: Escreva um algoritmo que imprima de 1 até 10. 
programa
{
	funcao inicio()
	{
		inteiro i

		para (i=1; i<=10; i++) {
			escreva  (i ," ")
		//Observe que o usuário  abriu chaves no "para", mas não fechou-as.			
	}
}

CERTO

 
//Descricao do programa: Escreva um algoritmo que imprima de 1 a 10. 
programa
{
	funcao inicio()
	{
		inteiro i

		para (i=1; i<=10; i++) {
			escreva  (i ," ")
		}			
	}
}

Erros:

    1. Usar sinal de atribuição (=) em vez de comparação (==), ou vice-versa.
    2. Utilização incorreta do ponto e virgula (;) .
    3. Utilizar parêntesis Descasados .
    4. Utilizar chaves Descasados .
    5. Utilizar variáveis não declaradas .
    6. Colocar espaço entre os dois caracteres que compõem os operadores ( ==, <=,>=,!=) .
    7. Inversão da ordem dos dois caracteres que compõem os operadores (<=,>=,!=) escrevendo (=<,=>,=!)
    8. Dividir um número inteiro por um numero real .
    9. Usar as aspas Descasadas num Printf .
    10. Utilizar as aspas Descasadas num Scanf .
    11. Errar a escrita de um comando .
    12. Esquecer o E comercial (&) no Scanf .
    13. Colocação das flags incorretas (%c,%f,%d,...) no Scanf ou Printf .
    14. Criação de ciclo infinito.
    15. Utilização de barra (/) em vez de barra invertida (\).
    16. Declaração de variável incompatível com o seu tipo declarado.
    17. Fazer comparação com uma variável do tipo char e um caracter e não colocar este caracter entre aspas simples.
    18. Uso incorreto das chaves em comandos de repetição.
    19. Utilização incorreta de variáveis contadoras ou acumuladoras em comandos de repetição (Esquecer de declarar, ...).
    20. Erro na declaração de variáveis, relacionados ao nome.
    21. Erros de Case Sensitive.
    22. Uso de código que não é necessário para a solução do problema.
    23. Código Incompleto.
    24. Erro Semântico
    25. Código não corresponde com a questão.
    26. Utilização de variável sem necessidade.
